# 📚 文档说明

本文件夹包含项目的所有文档文件。

## 📂 文件夹结构

```
docs/
├── README.md           # 本文件
├── archive/            # 归档文档（历史文档、临时文档）
└── architecture/       # 架构文档（如果存在）
```

## 📝 文档分类

### 当前使用的文档
- 应该放在 `docs/` 根目录
- 例如：`docs/API文档.md`

### 归档文档
- 历史文档、临时文档、已完成的报告
- 放在 `docs/archive/`
- 例如：`docs/archive/部署执行报告.md`

## ⚠️ 注意事项

1. **不要将文档放在根目录**
   - 所有 `.md` 文件都应该放在 `docs/` 文件夹中
   - 只有 `README.md` 可以放在根目录

2. **文档命名规范**
   - 使用有意义的名称
   - 避免使用特殊字符
   - 中文文档名也可以

3. **定期整理**
   - 完成的文档移动到 `archive/`
   - 删除过时的文档

## 🔄 如何添加新文档

1. **确定文档类型**
   - 当前使用的 → `docs/`
   - 历史/临时 → `docs/archive/`

2. **创建文档文件**
   - 在对应的文件夹中创建新文件
   - 不要放在根目录

3. **更新索引**
   - 如果是重要文档，更新相关 README
